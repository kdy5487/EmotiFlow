# EmotiFlow UI/UX 설계 문서 (AI 일기 중심)

본 문서는 EmotiFlow의 핵심 흐름(감정 기록 + AI 대화 + 시각화)을 중심으로 UI/UX 방향성을 정의합니다.

---

## 1. 목표
*   감정을 “기록 → 대화 → 시각화”로 자연스럽게 연결
*   대화 자체의 재미(캐릭터/질문 던전/감정 인터랙션) 제공
*   모바일/태블릿/웹에서 일관된 경험 제공

---

## 2. UX 핵심 플로우
1. 감정 선택 → 2. AI 대화 → 3. 일기 저장 → 4. 시각화 피드백

---

## 3. 정보 구조(IA) 초안
*   홈
*   일기
    - 목록 / 상세 / 작성 / AI 채팅 일기
*   AI
    - 분석 / 카드 / 패턴
*   설정

---

## 4. 화면별 UX 상세 (AI 일기 우선)

### 4.1 홈
*   **상단**: 오늘 감정 선택(이모지 1~2개)
*   **중단**: AI 대화 시작 CTA + 질문 카드 3~5개
*   **하단**: 오늘 감정 시각화 카드(레이어/스펙트럼 중 택1)
*   **의도**: "지금 바로 대화를 시작해야 하는 이유"를 명확히 제공

### 4.2 AI 대화 (핵심 차별화) ✅ **2026-01-21 개편 완료**
*   **구현 완료 사항:**
    - ✅ 감정별 캐릭터 프로필 시스템
      - 감정 선택 전: 대표 캐릭터 Emoti 표시
      - 감정 선택 후: 해당 감정 캐릭터 표시
    - ✅ 감정별 배경색 자동 변경 (부드러운 파스텔 톤)
    - ✅ 캐릭터 기반 감정 선택 UI
      - 4열 그리드 레이아웃
      - 캐릭터 이미지 + 감정명 표시
      - 선택 시 강조 효과 (테두리, 그림자)
    - ✅ 모던한 말풍선 디자인
      - AI: 흰색 배경, 왼쪽 상단 모서리만 각짐
      - 사용자: 보라색 배경, 오른쪽 상단 모서리만 각짐
      - 부드러운 그림자 효과
    - ✅ 타이핑 인디케이터 캐릭터 통합
    - ✅ 헤더에 현재 대화 캐릭터 미니 아이콘 표시
*   **추후 확장 예정 (우선순위):**
    - [ ] 캐릭터별 말투 차별화 (프롬프트 시스템 - 문서화 진행 중)
    - [ ] 캐릭터 선택 탭 (대화 중 캐릭터 변경)
    - [ ] 감정 온도계 슬라이더 (대화 중 강도 조절)
    - [ ] 질문 카드(던전 모드) 혼합
*   **현재 대화 영역:**
    - 기본 대화 + 캐릭터 프로필 통합
    - 자연스러운 스크롤
*   **하단 행동 영역:**
    - 감정 선택 UI (감정 미선택 시)
    - 메시지 입력창

### 4.3 일기 작성(일반)
*   감정 선택 + 강도 조절
*   미디어 첨부(사진/드로잉)
*   저장 후 시각화 피드백(미니 카드)

### 4.4 저장 완료 피드백
*   감정 시각화 카드 즉시 표시
*   “기록하면 보인다” 보상 루프 강화

## 5. AI 대화 재미 요소 상세(후보들임 아직 개발 x)
기본 ai와의 대화 베이스가 존재하고 아래의 것을 추가할 수 도있음. 
### 5.1 멀티 캐릭터 NPC
*   감정을 고른 캐릭터와 대화
*   캐릭터마다 기본적으로는 다 같은 프롬포트 적용
*   추후 캐릭터별 말투/문장 길이/이모지 스타일 차별화 or 캐릭터마다가 아닌 세션 시작 시 선택, 대화 중 변경 가능

### 5.2 질문 던전(카드 선택)
*   한 턴에 3~5개 질문 제공
*   난이도 표시: 가벼움 / 보통 / 깊음
*   카드 선택에 따른 간단 분기 구조

### 5.3 감정 온도계
*   대화 중 강도를 슬라이더로 조절
*   강도 변화에 따라 AI 답변 톤 조정
---

## 6. 시각화 방향성
*   **감정 레이어 캘린더**: 하루 감정을 층으로 쌓아 표현
*   **감정 스펙트럼 링**: 조합 비율을 원형으로 표현
*   **감정 씨앗 → 성장 카드**: 기록 누적이 성장으로 보이게 표현

---

## 7. 비주얼 시스템
*   **톤**: 미니멀 + 소프트 그라데이션 + 라운드 카드
*   **컬러**: 메인 퍼플 + 라일락 보조, 감정 컬러는 서브 톤
*   **타이포**: 전역 폰트 1종 + 고정 스케일(헤드라인/본문/캡션)
*   **아이콘/일러스트**: 캐릭터/감정 이모지 스타일 일관성 유지

---

## 8. 반응형/다크모드 기준
*   모든 화면에 다크모드 지원
*   브레이크포인트 기준: 모바일 / 태블릿 / 웹
*   웹에서는 좌우 여백 확장 + 카드 그리드화

---

## 9. 접근성/터치 영역
*   최소 터치 영역 44~48dp
*   대비 기준 준수
*   중요한 CTA는 색상+텍스트 동시에 강조

---

## 10. 빈 상태/오류 UX
*   빈 상태: 감정 기록 유도 문구 + CTA 제공
*   오류: 간결한 메시지 + 재시도 CTA

---

## 11. 애니메이션/모션
*   과한 모션 지양
*   감정 선택/저장 완료 시 미세한 피드백 모션 추가
*   대화 카드 선택 시 전환 애니메이션 적용
