# Gemini API 프롬프트 가이드

EmotiFlow 앱에서 사용하는 Google Gemini API 프롬프트들의 구조와 목적을 설명합니다.

## 🔄 최근 변경사항 (2026년 1월)

### 주요 개선사항
- **감정 선택 제한**: AI 대화 시 감정을 하나만 선택하도록 변경
- **프롬프트 간소화**: 초기 질문과 대화 프롬프트를 더 간결하게 개선
- **반복 방지**: AI가 비슷한 말을 반복하지 않도록 프롬프트 강화
- **응답 길이 최적화**: AI 응답을 3-4문장으로 제한하여 가독성 향상
- **자연스러운 대화**: 어색한 표현과 반복적인 내용 제거
- **맥락 유지 강화**: 이전 대화 내용을 참고하여 일관성 있는 응답 생성
- **상황별 대응**: 슬픔/기쁨/분노 등 감정에 따라 공감/위로/응원/해결방법 제시
- **과한 표현 제거**: "소중한", "온기가 전해지는" 등 과도한 수식어 금지

## 🎯 1. 초기 인사 프롬프트 (generateEmotionSelectionPrompt)

### 목적
사용자가 일기 작성을 시작할 때 자연스럽고 편안한 대화 시작을 유도

### 현재 적용 중인 프롬프트 (2026-01-21)
```dart
사용자가 오늘 하루를 기록하기 위해 들어왔습니다. 첫 인사를 간단하게 해주세요.

**규칙:**
1. 1-2문장으로 짧게 (최대 2문장)
2. 과한 표현 금지 ("소중한", "마음이 전해지는" 등)
3. 감정 선택이나 자유롭게 이야기할 수 있다고 간단히 안내
4. 이모지 사용 금지

**예시:**
- "안녕하세요. 오늘 하루는 어떠셨나요?"
- "오늘 특별히 기억에 남는 일이 있었나요?"

한국어로 답변해주세요.
```

### 설계 의도
- **간결함 우선**: 첫 인사는 짧고 명료하게 → 사용자 부담 최소화
- **과한 표현 제거**: "소중한", "온기가 전해지는" 등 감정 과잉 표현 금지
- **선택권 제시**: 감정 선택 또는 자유 대화 모두 가능하다는 안내로 유연성 제공
- **이모지 제거**: 텍스트 중심의 깔끔한 UI 유지

### Fallback 응답 (API 키 없을 때)
```dart
"안녕하세요. 오늘 하루는 어떠셨나요?"
```
- API 호출 실패 시에도 자연스러운 대화 시작 보장

## 💭 2. 대화 생성 프롬프트 (generateEmotionBasedQuestion)

### 목적
사용자가 선택한 감정과 응답을 바탕으로 맞춤형 대화를 생성하여 자연스러운 상담 경험 제공

### 현재 적용 중인 프롬프트 (2026-01-21)
```dart
당신은 사용자의 감정을 진심으로 이해하고 함께하는 대화 상대입니다.

**상황:**
- 사용자 감정: $selectedEmotion
- 사용자가 방금 말한 내용: "$userResponse"
- 이전 대화 맥락: ${userOnlyHistory.join(' | ')}
- 방금 전 당신이 한 말 (반복 금지): "$lastAiMessage"

**대화 원칙:**
1. **진짜 공감**: 감정을 중요하게 받아들이세요. "그렇군요" 같은 무미건조한 반응 금지.
2. **자연스러운 반응**: 상황에 맞게 반응하세요.
   - 슬픔/힘듦 → 공감 + 위로 (때로는 응원)
   - 기쁨/설렘 → 함께 기뻐하기
   - 분노/답답함 → 공감 + (필요시) 해결 방향 제안
3. **적당한 길이**: 3-4문장 (너무 짧거나 길지 않게)
4. **스마트한 질문**: 
   - 너무 세세한 건 묻지 마세요 (예: "어떤 면접이었나요?" → 어색함)
   - 감정이나 느낌 중심으로 질문하세요 (예: "그때 어떤 기분이 드셨어요?")
5. **다양한 반응**: 공감/위로/궁금증/응원/해결방법을 상황에 맞게 섞으세요.

**좋은 예시 (면접 탈락 - 슬픔):**
많이 실망스러우셨겠어요. 준비한 만큼 기대도 컸을 텐데 정말 속상하셨을 것 같아요.
그래도 면접까지 가신 것 자체가 대단한 일이에요. 이번에 배운 점이 있다면 무엇인가요?

**나쁜 예시:**
그렇군요. 어떤 면접이었나요? 힘내세요!

**금지사항:**
- "그렇군요" + "질문" 패턴 반복 금지
- "힘내세요", "화이팅" 같은 형식적 응원 금지
- 마크다운 표시 금지
- 이전 응답과 70% 이상 유사한 문장 사용 금지

한국어로 답변해주세요.
```

### 설계 의도
- **맥락 유지**: 이전 대화 내용을 분석하여 일관성 있는 대화 유지
- **반복 방지**: 
  - 직전 AI 응답을 프롬프트에 포함 → "이전 말과 다르게" 명시
  - "그렇군요 + 질문" 패턴 금지 → 딱딱한 느낌 제거
- **상황별 대응**: 감정에 따라 공감/위로/응원/해결방법을 다르게 제시
- **자연스러운 질문**: 
  - 너무 세세한 질문(어떤 면접?) 금지 → 어색함 방지
  - 감정 중심 질문(어떤 기분?) → 대화 깊이 증가
- **적절한 길이**: 3-4문장 → 너무 짧지도, 길지도 않게

### Fallback 응답 (API 키 없을 때)
```dart
"$userResponse에 대해 더 자세히 말씀해주시겠어요?"
```
- 사용자 응답을 반영한 최소한의 자연스러운 질문 생성

- **UI 단순화**: 감정 선택 UI를 더 직관적이고 사용하기 쉽게 개선

### 코드 변경사항

// 변경 전

## 🤗 감정 분석 및 위로 프롬프트

### 목적
사용자의 감정을 분석하고 공감과 위로를 제공하며 실용적인 조언 제시

### 프롬프트 구조
```
사용자와의 대화 내용: {대화_전체}
선택된 감정: {감정}

다음 내용을 자연스럽게 통합하여 응답해줘:

1. **감정 인정과 공감**
   - 사용자의 감정을 있는 그대로 인정
   - "그럴 수 있어요", "충분히 이해해요" 등의 공감 표현

2. **따뜻한 위로**
   - 사용자의 상황에 맞는 따뜻한 위로
   - 감정의 정당성 인정

3. **실용적 조언**
   - 구체적이고 실행 가능한 1-2가지 조언
   - 사용자의 상황에 맞는 맞춤형 해결책

**응답 스타일:**
- 분석, 공감, 위로, 조언을 자연스럽게 하나로 통합
- 5-6문장을 넘지 않도록 간결하게
- 1-2가지 실용적인 팁 제공
- 계속 대화할 수 있도록 격려

**주의사항:**
- 각 섹션을 명확히 구분하지 말고 자연스럽게 연결
- 비슷한 내용의 반복적인 답변 피하기
- "힘내세요" 같은 마무리보다는 지속적인 대화 유도
```

## 📝 3. 일기 요약 생성 프롬프트 (generateDiarySummary)

### 목적
대화 내용을 바탕으로 자연스러운 1인칭 일기로 정리하고 가독성 있게 제시

### 현재 적용 중인 프롬프트 (2026-01-21)
```dart
사용자와 나눈 대화를 바탕으로 일기를 완성해주세요.

**대화 내용:**
${conversationHistory.join('\n')}

**감정:** $selectedEmotion

**핵심 원칙 (반드시 지켜주세요):**
1. **사실만 기록**: 사용자가 말한 내용만 작성하세요. 추측이나 과장 금지.
2. **1인칭 시점**: "나는", "내가" 등 사용자가 직접 쓴 것처럼.
3. **자연스러운 일기체**: 진솔하고 편안한 말투로.
4. **가독성**: 2-3문장마다 줄바꿈을 넣어주세요.

**길이 (대화량에 비례):**
${isShortConversation ? 
  '- 짧은 대화 → 4-6문장 (간결하게)\n- 무리하게 늘리지 마세요.' : 
  '- 충분한 대화 → 6-10문장\n- 대화 내용을 충실히 반영하세요.'}

**금지사항:**
- "아릿한", "포근한", "온기가 전해지는" 등 시적 표현 사용 금지
- 사용자가 말하지 않은 내용 추가 금지
- "제목:", "내용:" 같은 라벨 금지

**출력 형식:**
[일기 본문만 작성, 제목이나 라벨 없이]

한국어로 자연스러운 일기체로 작성해주세요.
```

### 설계 의도
- **사실 중심**: AI가 과장하거나 추측하지 않도록 → 사용자 신뢰 유지
- **1인칭 시점**: "나는", "내가" → 사용자가 직접 쓴 것처럼 자연스러움
- **가독성 강화**: 
  - 2-3문장마다 줄바꿈 → 긴 텍스트 블록 방지
  - 시적 표현 금지 → "온기가 전해지는" 같은 과한 수식어 제거
- **대화량 비례**: 
  - 짧은 대화(5턴 미만) → 4-6문장으로 간결하게
  - 충분한 대화(5턴 이상) → 6-10문장으로 충실하게
  - 무리하게 늘리지 않음 → 자연스러움 유지
- **라벨 제거**: "제목:", "내용:" 같은 구조적 표시 금지 → 순수 일기만 생성

### Fallback 응답 (API 키 없을 때)
```dart
"오늘은 ${selectedEmotion}을(를) 느낀 하루였다.\n\n" +
(대화 내용 중 사용자 메시지만 추출하여 간단히 정리)
```
- 대화 내용이 있더라도 최소한의 일기 형태로 제공

## 🎨 AI 이미지 생성 프롬프트

### 목적
사용자의 대화 내용과 감정을 바탕으로 맞춤형 이미지 생성을 위한 상세 프롬프트 작성

### 프롬프트 구조 (실제 구현)
```
다음 내용을 바탕으로 감정적이고 아름다운 일기 그림을 그려주세요:

감정: {선택된_감정}
일기 내용: {일기_요약}
핵심 키워드: {추출된_키워드}

**스타일 가이드:**
- 감정에 맞는 색감과 분위기
- 일기 내용을 상징적으로 표현
- 따뜻하고 아름다운 일러스트레이션
- 한국적인 감성과 현대적인 디자인

이 그림은 사용자의 개인적인 감정과 경험을 표현하는 일기용 이미지입니다.
```

### 엔드포인트/파라미터 (구현 기준)
- 모델: `gemini-1.5-flash:generateContent`
- 요청 바디: `contents.parts[].text`에 프롬프트 전달
- generationConfig: `temperature: 0.8`, `maxOutputTokens: 2048`
- 반환 처리: 첫 candidate → content.parts[0].text 추출(텍스트 URL 가정) → `Image.network` 미리보기
- 실패 처리: null 반환 → 다이얼로그/스낵바 안내

### 키워드 추출 로직
```javascript
// 감정 관련 키워드
if (message.contains('기쁨') || message.contains('행복') || message.contains('즐거')) 
    keywords.add('기쁨');

// 활동 관련 키워드  
if (message.contains('산책') || message.contains('걷기')) 
    keywords.add('산책');
if (message.contains('음식') || message.contains('밥') || message.contains('먹')) 
    keywords.add('음식');
if (message.contains('친구') || message.contains('사람') || message.contains('만남')) 
    keywords.add('사람');
```

## 💡 프롬프트 최적화 팁

### 1. 응답 길이 제어
- **질문**: 1-2문장으로 간결하게
- **위로/조언**: 2-3문장으로 충분한 내용 제공
- **요약**: 6-7문장 이내로 제한

### 2. 자연스러운 대화 흐름
- 각 섹션을 명확히 구분하지 말고 자연스럽게 연결
- 마크다운(`**`, `##`) 사용 금지
- 과도한 이모지 사용 금지

### 3. 사용자 맞춤화
- 대화 기록을 활용한 개인화된 응답
- 사용자의 감정 상태에 따른 적절한 톤 조절
- 이전 대화와의 연결성 유지

### 4. 한국어 특성 고려
- 존댓말 사용으로 정중하고 따뜻한 느낌
- 한국 문화에 맞는 표현과 예시 사용
- 감정 표현의 미묘한 차이 반영

### 5. API 파라미터 최적화
```javascript
generationConfig: {
  temperature: 0.8,        // 창의성과 일관성의 균형
  topK: 40,               // 응답의 다양성 제어
  topP: 0.95,             // 높은 품질의 응답 보장
  maxOutputTokens: 2048,  // 충분한 응답 길이 확보
}
```

## 🏗️ 프롬프트 설계 원칙 및 향후 계획

### 핵심 설계 원칙

#### 1. 명확성 (Clarity)
- **구체적인 지시**: "간결하게" → "3-4문장으로"
- **예시 제공**: 좋은 응답/나쁜 응답 예시 명시
- **금지사항 명시**: "~하지 마세요" 형태로 구체적 제한

#### 2. 맥락 유지 (Context Awareness)
- **대화 히스토리 활용**: 이전 대화 8턴까지 참조
- **직전 AI 응답 포함**: 반복 방지를 위해 "방금 한 말"을 프롬프트에 포함
- **사용자 감정 추적**: 선택한 감정을 모든 프롬프트에 명시

#### 3. 자연스러움 (Naturalness)
- **과한 표현 금지**: "소중한", "온기가 전해지는" 등 제한
- **형식적 표현 금지**: "힘내세요", "화이팅" 등 제한
- **1인칭 시점 유지**: 일기는 "나는", "내가"로 작성

#### 4. 적응성 (Adaptability)
- **상황별 대응**: 슬픔 → 위로, 기쁨 → 축하, 분노 → 공감+해결방안
- **대화량 비례**: 짧은 대화 → 간결한 요약, 긴 대화 → 상세한 요약
- **질문의 깊이 조절**: 세세한 질문(어떤 면접?) 금지, 감정 중심 질문 권장

#### 5. 안정성 (Reliability)
- **Fallback 전략**: API 키 없음/네트워크 오류 시 대체 응답 제공
- **모델 Fallback**: 주 모델 실패 시 다른 모델로 자동 전환
- **에러 핸들링**: 빈 응답/구조 오류 시 안전한 기본값 제공

### 향후 프롬프트 작성 가이드라인

#### 새 프롬프트 추가 시 체크리스트
1. **목적 명확화**: 이 프롬프트가 해결하려는 문제는 무엇인가?
2. **구체적 지시**: 응답 길이, 톤, 형식을 명확히 지정했는가?
3. **예시 제공**: 좋은/나쁜 응답 예시를 포함했는가?
4. **금지사항 명시**: 하지 말아야 할 것을 구체적으로 나열했는가?
5. **테스트**: 최소 5개 이상의 다양한 시나리오로 테스트했는가?
6. **Fallback**: API 실패 시 대체 응답을 준비했는가?

#### 프롬프트 개선 프로세스
1. **사용자 피드백 수집**: 실제 사용 중 불만족스러운 응답 기록
2. **패턴 분석**: 어떤 상황에서 문제가 발생하는가?
3. **프롬프트 수정**: 문제 패턴을 명시적으로 금지하거나 개선
4. **A/B 테스트**: 기존 vs 새 프롬프트 비교 (가능하면)
5. **문서화**: `gemini_prompts_guide.md`에 변경 이유와 내용 기록
6. **버전 관리**: Git 커밋 메시지에 프롬프트 변경 사항 명시

#### 향후 추가 예정 프롬프트
1. **감정 패턴 분석 프롬프트**
   - 목적: 최근 일기에서 감정 흐름 요약
   - 예: "이번 주는 기쁨 → 슬픔 → 평온 순서로 감정 변화가 있었습니다."

2. **행동 제안 프롬프트**
   - 목적: 감정에 맞는 구체적 실천 방안 1-3개 제시
   - 예: "슬픔을 느끼셨다면: 1) 좋아하는 음악 듣기 2) 친구에게 연락하기"

3. **감정 회고 카드 프롬프트**
   - 목적: 대화 내용을 바탕으로 "오늘의 한 문장" 생성
   - 예: "면접에서 떨어졌지만, 준비 과정에서 많은 것을 배웠다."

4. **NPC 캐릭터별 프롬프트**
   - 목적: 따뜻한/직설적인/유쾌한 캐릭터별 말투 구현
   - 예: 유쾌한 캐릭터 → "어머, 그랬구나! 그래서 어떻게 됐어?"

### API 파라미터 최적화 전략

#### 현재 설정 (2026-01-21)
```javascript
generationConfig: {
  temperature: 0.8,        // 창의성과 일관성의 균형
  topK: 40,               // 응답의 다양성 제어
  topP: 0.95,             // 높은 품질의 응답 보장
  maxOutputTokens: 2048,  // 충분한 응답 길이 확보
}
```

#### 파라미터별 설명
- **temperature (0.8)**: 
  - 0에 가까울수록 일관적/예측 가능한 응답
  - 1에 가까울수록 창의적/다양한 응답
  - 0.8 선택 이유: 상담은 일관성과 창의성 모두 필요
  
- **topK (40)**: 
  - 각 단계에서 고려할 단어 개수
  - 낮을수록 안정적, 높을수록 다양함
  - 40 선택 이유: 충분한 다양성 확보
  
- **topP (0.95)**: 
  - 누적 확률 임계값
  - 높을수록 더 많은 단어 고려
  - 0.95 선택 이유: 품질 유지하면서 다양성 확보
  
- **maxOutputTokens (2048)**: 
  - 최대 응답 길이
  - 2048 선택 이유: 충분한 일기 요약 가능

#### 향후 최적화 방향
- **초기 인사**: temperature 0.6 (더 일관적으로)
- **대화**: temperature 0.8 (현재 유지)
- **일기 요약**: temperature 0.7 (사실 중심, 약간 낮춤)

## 📊 성능 모니터링 및 평가

### 주요 지표

#### 1. 응답 시간
- **목표**: 평균 2-3초 이내
- **측정 방법**: API 호출 시작~응답 수신까지의 시간
- **현재 상태**: Gemini Flash 모델 사용으로 2초 내외 달성
- **개선 방향**: 
  - 프롬프트 길이 최적화 (불필요한 설명 제거)
  - 캐싱 전략 도입 (동일 감정에 대한 초기 질문)

#### 2. 응답 품질
- **평가 기준**:
  - 맥락 일치도: 이전 대화 내용을 적절히 반영하는가?
  - 자연스러움: 과한 표현이나 어색한 질문이 없는가?
  - 반복 방지: 비슷한 응답을 반복하지 않는가?
  - 적절한 길이: 너무 짧거나 길지 않은가?
- **측정 방법**: 
  - 사용자 피드백 (좋아요/싫어요 버튼 - 향후 추가 예정)
  - 수동 샘플링 및 평가 (주간 10개 대화 검토)
- **현재 상태**: v3.0 프롬프트 적용 후 자연스러움 개선됨

#### 3. API 사용량
- **목표**: Gemini API 무료 한도 내 운영
- **무료 한도**:
  - Gemini 1.5 Flash: 분당 15 RPM (Requests Per Minute)
  - 일일 1500 RPD (Requests Per Day)
- **관리 전략**:
  - Fallback 응답 우선 활용 (API 키 없을 때)
  - 불필요한 API 호출 최소화 (캐싱)
  - 사용량 모니터링 및 경고 시스템 (향후 추가)

#### 4. 오류율
- **목표**: 5% 이하 유지
- **오류 유형**:
  - API 키 없음 → Fallback으로 처리 (정상)
  - 네트워크 오류 → Fallback으로 처리 (정상)
  - 모델 404 → 다른 모델로 자동 전환 (정상)
  - 빈 응답 → Fallback으로 처리 (정상)
- **현재 상태**: 모델 Fallback 체계로 안정성 확보

### 개선 포인트 및 Action Items

#### 단기 (1-2주)
1. **프롬프트 길이 최적화**
   - 현재: 긴 설명문 포함 → 토큰 소비 증가
   - 개선: 핵심 지시만 남기고 예시 축소
   - 효과: 응답 시간 10-20% 단축 예상

2. **반복 응답 모니터링**
   - 현재: 수동 확인
   - 개선: 응답 유사도 자동 측정 로직 추가
   - 효과: 반복 패턴 조기 발견

3. **사용자 피드백 수집**
   - 현재: 피드백 시스템 없음
   - 개선: AI 응답마다 "도움이 됐나요?" 버튼 추가
   - 효과: 데이터 기반 프롬프트 개선

#### 중기 (1-2개월)
1. **캐싱 전략 도입**
   - 초기 인사 응답 캐싱 (매번 동일하므로)
   - 감정별 첫 질문 템플릿 캐싱
   - 효과: API 호출 20-30% 감소 예상

2. **A/B 테스트**
   - 프롬프트 변경 시 이전 버전과 비교
   - 사용자 만족도 차이 측정
   - 효과: 객관적 개선 검증

3. **다국어 지원 준비**
   - 프롬프트 구조 다국어 확장 가능하도록 설계
   - 한국어 → 영어 번역 테스트
   - 효과: 글로벌 사용자 대응

#### 장기 (3개월+)
1. **Fine-tuning 고려**
   - Gemini API의 Fine-tuning 기능 활용
   - EmotiFlow 스타일에 맞춘 맞춤형 모델 구축
   - 효과: 프롬프트 길이 단축, 품질 향상

2. **Multi-turn 대화 최적화**
   - 현재: 전체 히스토리 전송 → 토큰 낭비
   - 개선: 요약된 맥락만 전송
   - 효과: 긴 대화에서 비용 절감

3. **실시간 감정 분석**
   - 사용자 응답에서 감정 변화 자동 감지
   - 감정 변화 시 대응 전략 변경
   - 효과: 더 민감한 상담 경험
