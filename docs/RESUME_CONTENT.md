## 프로젝트 개요

**프로젝트명**: EmotiFlow - AI 기반 감정 일기 앱  
**개발 기간**: 2025.08.04 - 2025.08.19 (2주)  
**역할**: 단독 개발 (기획, 설계, 개발, 배포 전 과정)  
**기술 스택**: Flutter, Dart, Firebase, Google Gemini API, Riverpod, GoRouter

---

## 프로젝트 설명

AI와 대화하며 감정을 기록하고, 감정 분석을 해주는 Flutter 기반 감정 일기 앱입니다. 사용자의 일상적인 감정을 AI와 함께 분석하고, 개인화된 인사이트를 제공하여 감정 관리와 성장을 돕습니다.

---

## 주요 기능

### 1. AI 대화형 일기 작성
- Google Gemini 2.5 Flash API 연동
- AI가 질문을 던지고 사용자가 응답하며 자연스럽게 일기를 완성하는 대화형 UX
- 감정 기반 맞춤형 질문 생성
- 실시간 대화형 인터페이스

### 2. 자유형 일기 작성
- 텍스트·이미지를 자유롭게 기록하는 기본 일기 작성 기능
- 감정 선택 및 강도 설정 (10가지 감정)
- 그림 그리기 캔버스 (스티커, 도형 도구 포함)

### 3. 일기 목록 관리
- Firestore 실시간 동기화 기반 검색 / 필터 / 정렬
- 리스트/그리드 뷰 전환
- 일괄 삭제 모드

### 4. AI 감정 분석 및 조언
- 작성된 일기를 분석해 감정 요약과 조언 제공
- 개인화된 조언 생성
- 감정 트렌드 차트 시각화

### 5. 사용자 인증 및 프로필
- Firebase Authentication (Google 로그인)
- 프로필 관리 시스템
- Material 3 기반 다크/라이트 테마 지원

- 전체 프로젝트 기획 및 설계

- Firebase 연동 (Auth, Firestore, Storage)

## 주요 성과

- **Clean Architecture 적용**: Domain/Data/Presentation 계층 분리로 확장성 확보
- **성능 최적화**: UI 지연 시간 95% 개선 (2200ms → 50ms)
- **코드 품질 개선**: 대규모 리팩토링으로 파일 크기 50% 감소 (1005줄 → 500줄)
- **실시간 데이터 동기화**: Firestore Stream과 UI 상태 분리로 안정적인 UX 구현
- **아키텍처 설계**: 데이터 상태와 UI 상태 분리 전략 수립

## 사진/스크린샷 처리 방법

### 추천 방법
1. **앱 스크린샷**: 주요 화면 3-5개 (홈, 일기 작성, AI 채팅, 일기 목록)
2. **아키텍처 다이어그램**: Clean Architecture 계층 구조 (직접 그리거나 다이어그램 도구 사용)
3. **트러블슈팅 미리보기**: 대표 트러블슈팅 1개를 간단히 요약

### 트러블슈팅 미리보기 예시

**감정 선택 UI 지연 문제 (95% 성능 개선)**
- 문제: AI 채팅 페이지 진입 시 2-3초 지연
- 해결: Fallback 즉시 표시 + 비동기 API 호출
- 결과: 2200ms → 50ms (95% 개선)
